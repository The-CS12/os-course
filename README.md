## آموزش به سبک تٍرِد 

تِرِد ها پست های کوتاه با هدف بیان یک ایده به سریع ترین حالت ممکن هستند... حالا اگر هر مطلب آموزشی رو به شکلی ریز ریز کنیم که بتونیم در واحد های اولیه ای به عنوان تِرِد آموزششون بدیم میتونیم مخاطب رو خسته نکنیم و از این قضیه برای آموزش هر مطلبی به شکلی بهتر استفاده کنیم...

### نحوه شروع کار با `NTB-THEME`

این تمپلیت به صورت خاص برای اعضای کامیونیتی `CS12` طراحی شده برای شروع کار کافیست که :

1. پروژه رو فورک کنید 
2. سپس پروژه فورک شده رو clone کنید
3. به دایرکتوری پروژه برید
4. مطمعن شوید که نرم افزار hugo extended رو روی سیستم نصب دارید اگر ندارید از اینجا [نصب کنید](https://gohugo.io/installation/)
5. فایل hugo.toml رو پیدا کنید
6. آن را ویرایش کنید
7. به دایرکتوری `content/threads` برید
8. یک دایرکتوری برای chapter اول خود با نام دلخواه بسازید
9. یک فایل با نام `index.md_`  در آن دایرکتوری ایجاد کنید.
10. محتویات فایل را باید چنین باشد : 
```
+++

title = "عنوان فصل"

# وضعیت فصل :
# done -> پایان
# running -> در حال اجرا

group_status = "done"

# وزن هر فصل هر چه مقدار کمتری باشه بالا تر نمایش داده میشه...
# مثلا برای فصل 1 باید 1 باشد و برای فصل 2 باید 2 و ...
weight = 1

+++
```

<span style="background:#f003; font-weight:600; display:block;"> توجه داشته باشید</span>
> مقدار weight از 1 شروع میشود مقدار 0 قبلا برای فصل site که وجودش ضروری هست رزرو شده است.

11. سپس برای ساخت پست یک پست کافیست از دستور زیر استفاده کنید :
```
hugo new threads/<chapter-dir>/<nameOfPost>.md
```

- `<chapter-dir>` : همان دایرکتوری ای هست که در threads ساختید
- ‍‍`<nameOfPost>` : نام فایل و پست شماست 

پیشنهاد میکنم نام پست ها رو به صورت عددی بگذارید برای مثال 01 اینطوری بعدا مدیریت thread ها و ordering شون خیلی راحت تر خواهد بود...

```
hugo new hugo new threads/<chapter-dir>/01.md
```

12. سپس پستی با این محتوا ساخته میشه: 
‍‍‍
```
---

# عنوان پست

title: "02"

# تاریخ انتشار پست

date: 2026-06-18T13:27:33+03:30

# اگر فالس باشه نمایش داده نمیشه

draft: true

# اوردر پست بهتره به صورت اینکریمنتال افزایش پیدا کنه

order: 1

# توضیح کوتاه پست برای seo

description: ""

# تصویر شاخص پست فقط کافیه که نام تصویر رو بزارید
# اول تصویر رو در /static/assets/img قرار بدید
# سپس نام تصویر رو اینجا بگزارید مثلا x.jpeg
image: ""

# اگر میخوای این مطلب کامنت داشته باشه...

comments: true

---

<!-- کد ریوزیبل بخش یکم بیشتر بدونید -->

  

<!-- به صورت پیشفرض اوپن فالس هست تا کاربر برای دیدن محتوا کلیک کنه -->

<!-- اگر میخواستی به صورت پیشفرض باز باشه true بزارش -->

<!-- title به صورت پیشفرض همون یکم بیشتر بدونید هست میتونی پاکش کنی... -->

  

{{< readmore title="یکم بیشتر بدونید" open="false" >}}

متن کامل

{{< /readmore >}}

  

<!-- کد ریوزیبل مودال مدیا وسط پست -->

  

{{<

    mediamodal type="تایپ رو بنویس"

    text="متن مودال"
```

که کامنت ها گویای مطلب هستند.... اگر سوالی داشتید میتونید issue بزنید.

13. پس از نوشتن پست کافیه که دستور زیر رو بزنید و در محیط لوکال پستتون رو ببینید : 
‍‍‍
```
hugo serve -D
```

14. اگر اوکی بود پراسس رو kill کنید و دستور
‍‍
```
hugo --gc --minify
```

رو بزنید ... این نسخه نهایی آماده پابلیش رو برای شما میسازه...

15. حاله کافیه که تغییرات رو push کنید و از طریق github pages یا vercel و یا هر پلتفرمی که سایت های استاتیک رو میتونه هاست کنه دیپلوی کنید... توجه کنید که مقدار baseURL در hugo.toml رو باید برابر با دامنه بگذارید... تا دکمه های share و ... کار کنند.

باقی موارد در سورس پروژه و در جایی که نیاز بوده به صورت کامنت برای شما نوشته شده کل کاری که باید بکنید اینه که hugo.toml رو بخونید...
