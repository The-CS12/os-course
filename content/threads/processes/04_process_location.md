هر Process به طور فیزیکی یه **Process image** محسوب میشه! البته اشتباه نکنید منظور از ایمیج فایل های ایمیج یا **iso** نیستن بلکه به تمامی اطلاعاتی که برای اجرای یه پراسس نیاز داریم رو یک Process image میگیم. در واقع هر پراسس شامل کلی دیتا میشه! برای همین این اطلاعات رو یه Image در نظر میگیرن. 📄 <br>
✍️ **استالینگز** با این جمله این قضیه رو خیلی خوب منتقل میکنه :‌ <br>
> **تجلی فیزیکی یک Process چیزی به نام Process image است.**

در واقع این دیتا ها در قالب یک ایمیج ذخیره نمیشه بلکه با کنار هم قرار گیری دیتا های مربوط به هر Process یک ایمیج فرضی میسازن!‌‌‌ (امیدوارم تونسته باشم منظورمو منتقل کنم 😁) <br>

اجزای Process image :
- User program (ماشین کد برنامه)
- User data (متغیر های Global, heap ,...)
- Stack (مقادیر برای مدیریت اجرای توابع )
- Process control block (`PCB`)
