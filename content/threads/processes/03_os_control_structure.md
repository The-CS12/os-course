به نظرتون سیستم عامل برای کنترل پراسس ها و مدیریت منابع به چه اطلاعاتی نیازی داره؟🤔 <br>

اگر قرار باشه سیستم عامل تمامی پراسس ها رو کنترل، منابع رو مدیریت و خلاصه حواسش به همه چی باشه پس باید درباره تمام اون ها اطلاعات داشته باشه! سیستم عامل چندین ساختار **(Structure)** برای ذخیره سازی اطلاعات درباره منابع و پراسس ها داره. این ساختار ها عمدتا به صورت جدولی هستن.
- **Memory table 💭**
- **I/O table 💾**
- **File table 📂**
- **Process table 🧠**

اطلاعاتی که Memory table ذخیره میکنه شامل مقدار اختصاص داده شده از real memory و virtual memory به پراسس ها، ویژگی های امنیتی مثل قابلیت Shared memory و سایر اطلاعاتی که برای مدیریت مموری نیاز میشه هستش!
<br>
<br>

I/O table اطلاعاتی از جمله: نوع دیوایس I/O، وضعیت دسترسیشون، اینکه آیا در حال استفاده توسط Process خاصی هستن یا نه، آدرس مموری در حال استفاده و اطلاعات پراسسی که داره ازش استفاده میکنه توش دخیره میشه!
<br>
<br>

توی File table اطلاعاتی از جمله: وجودیت فایل ها، مکان و آدرسشون، وضعیتشون و سایر ویژگی ها مثل مالکان و سطح دسترسی اون ها ذخیره میشه!
{{< readmore type="warning" title="هشدار" open="false" >}}
توجه داشته باشید که File table با مفهوم File system تفاوت داره! در فصل های آینده به طور مفصل به فایل سیستم ها میپردازیم.
{{< /readmore >}}
<br>

و اما Process table که یکی از مهمترین ساختار های اطلاعاتی سیستم عامل هستش. اطلاعات همه پراسس ها توی این جدول ذخیره میشه. هر سطر از این جدول در واقع یک PCB هستش!!!‌ بله این جدول برای ذخیره سازی ‍`PCB` ها استفاده میشه که محل ذخیره سازی این جدول هم همون **Kernel space** هست!

{{< readmore type="" title="یکم بیشتر بدونید" open="true" >}}
PCB ها تمامی اطلاعات هر پراسس رو توی خودشون نگه نمیدارن. بلکه اطلاعات مهمی مثل Memory یا File رو با استفاده از Pointer (اشاره گر) ها به سایر Table ها لینک میکنن. <br>
> **بنابراین Process های توی Process table به سایر تیبل ها لینک دارن❗**
